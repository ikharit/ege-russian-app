## Дополнение: TeacherAnalytics fix — пустая глобальная статистика (2026-06-26, 17:56)

### Проблема
Страница "Аналитика всех пользователей" (`TeacherAnalytics.tsx`) показывала пустое состояние "Пока нет данных пользователей", даже когда demo-данные (`defaultTeacherStudents`) были доступны. Причина: `fetchAllUsers()` из `teacherAnalyticsStore.ts` возвращал пустой массив, если в Supabase `user_progress` не было записей (пользователь не авторизован или данные не синхронизированы). Не было fallback.

### Решение
- `TeacherAnalytics.tsx`: добавлен импорт `defaultTeacherStudents` из `gamificationSlice.ts`
- Добавлен `isDemo = !hasRealStudents && !realLoading` — флаг demo-режима
- `students` теперь имеет fallback: при отсутствии `realStudents` используется `defaultTeacherStudents`, маппится в `ProgressData` через `rawToProgressData()`
- Убран пустой экран `!hasRealStudents` — теперь всегда рендерится аналитика
- Добавлен жёлтый бейдж "Демо-данные" с пояснением, когда показывается fallback
- Расширенные метрики (`avgExamScore`, `avgTimeMinutes`, `avgMaxCombo`) остаются 0 при demo — это корректно, так как они зависят от `realStudents`

### Git
- `def4723` — fix(teacher-analytics): fallback to demo data when Supabase user_progress is empty
