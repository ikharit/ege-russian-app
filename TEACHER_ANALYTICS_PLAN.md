# План улучшений Teacher Analytics (Agent 2)

**Дата:** 2026-06-30  
**Цель:** Превратить Teacher Analytics в супер-информативный дашборд для учителя с детальной индивидуальной и массовой аналитикой.

---

## Фаза 1: Исправление критических багов с данными

### 1.1 `studentAnalytics.ts` — `lastAttemptAt` не существует в Supabase
**Проблема:** Строка 116 использует `taskStats[a].lastAttemptAt`, но Supabase `taskStats` имеет структуру `{total, correct, wrong}` без `lastAttemptAt`. При `lastActivityDate = undefined` код падает на `new Date(undefined)`.

**Fix:** Убрать fallback на `taskStats.lastAttemptAt`. Использовать только `stats.lastActivityDate`. Если `lastActivityDate` не установлен — `lastActivityDays = 999` (уже так работает через `daysSince`).

### 1.2 `detectPlayerType` — захардкоженные 0
**Проблема:** В `analyzeStudent` вызывается `detectPlayerType` с `leaderboardEntries=0, duelCount=0, challengeCount=0, classJoined=false`. Все ученики получают одинаковый тип.

**Fix:** Добавить поля в `UserStats` или использовать `rawProgressData` для получения реальных данных. Если данных нет — использовать randomized тип или оставить achiever как default.

### 1.3 `TeacherAnalytics.tsx` — `rawToProgressData` корректность
**Проверка:** `rawToProgressData` берёт `behaviorProfile` из `raw.behaviorProfile`. В `fetchAllUsers` `rawProgressData` уже содержит `behaviorProfile` из `admin_user_analytics`. ✅ Это работает.

## Фаза 2: Улучшения массовой аналитики (class-level)

### 2.1 Поиск, сортировка и фильтрация учеников
- **Поиск** по имени (input field)
- **Сортировка** по: точности, активности, риску, XP, прогрессу, имени
- **Фильтры** по: риску (low/medium/high), типу мотивации, диапазону точности

### 2.2 Дельта-метрики (week-over-week)
- Для каждого summary показывать: `+/- X%` по сравнению с 7 дней назад
- Источник: `dailySnapshots` из `admin_user_analytics`
- Новые карточки: "Прирост точности", "Прирост активности"

### 2.3 Корреляция — время vs точность
- Scatter chart: X = время в приложении (мин), Y = точность (%)
- Показывает: ученики, которые тратят много времени, но имеют низкую точность (inefficient learners)
- Цвет = риск

### 2.4 Прогноз ЕГЭ (Predictive Score)
- Подключить `predict()` из `predictiveScore.ts` для каждого ученика
- Показать: средний прогнозируемый балл класса, распределение (0-20, 21-40, 41-58)
- Список учеников с прогнозом < 30 — критично

### 2.5 Улучшенный heatmap — задания × ученики
- Matrix: строки = ученики, столбцы = задания (1-26)
- Цвет = точность (зелёный/жёлтый/красный)
- Быстро видеть: кто и какие задания не освоил

### 2.6 Когортный анализ (retention)
- Группировка учеников по дате первой активности (неделя)
- Для каждой когорты: сколько вернулись через 7, 14, 30 дней
- График retention curve

## Фаза 3: Улучшения индивидуальной аналитики (per-student)

### 3.1 Индивидуальный тренд в раскрывающейся карточке
- При expand показывать: line chart точности по дням (из `dailySnapshots`)
- График XP по дням
- График времени по дням

### 3.2 Таймлайн активности (GitHub-style)
- Календарь: 30×7 сетка, цвет = количество ответов/время
- Быстро видеть: когда ученик был активен, какие дни пропускал

### 3.3 Детали по заданиям (per-student)
- В раскрывающейся карточке: bars для каждого задания (1-26)
- Показать: точность, количество ответов, тренд (последние 7 дней)
- Сортировка: сначала самые проблемные

### 3.4 Сравнение учеников
- Чекбоксы на карточках учеников
- "Сравнить выбранных" → side-by-side таблица
- Сравнение: точность, прогресс, стрик, тип мотивации, слабые темы

### 3.5 История ошибок (per-student)
- В раскрывающейся карточке: последние 10 ошибок с explanation
- Сгруппировано по заданиям
- Ссылки на соответствующие уроки
