## Файлы

### 1. task10_atomized.xlsx
**Основной файл для импорта в приложение.**

#### Схема столбцов

| Столбец | Тип | Описание | Пример |
|---------|-----|----------|--------|
| `id` | int | Уникальный ID | 1 |
| `word_raw` | str | Слово с пропуском (как в сборнике) | "пр..исполнен" |
| `context` | str | Контекст в скобках | "отваги" |
| `correct_letter` | str | Правильная буква | "е" |
| `full_word` | str | Полное слово | "преисполнен" |
| `main_category` | str | Основная категория | "ПРЕ-/ПРИ-" |
| `subcategory` | str | Подкатегория атомизации | "ПРЕ- (правило значения)" |
| `rule` | str | Правило/пояснение | "ПРЕ- = 'до, перед, очень'" |
| `cluster` | str | Кластер созвучных слов | "преисполнен, превысить, превосходить" |
| `source` | str | Источник | "Дощинский-2026" |
| `difficulty` | int | Сложность 1-5 (опционально) | 3 |

#### Возможные значения main_category
- `ПРЕ-/ПРИ-`
- `З-/С-`
- `Ы-/И-`
- `Ъ-/Ь-`
- `РА-/РО- / РАЗ-/РОЗ- / РАС-/РОС-`
- `ПРА- / ПРО-`
- `Неизменяемые приставки`
- `Другое`

- `ПРЕ- (словарное)`
- `ПРИ- (правило значения)`
- `З- перед звонкой/гласной`
- `С- перед глухой`
- `С- неизменяемая`
- `Ы после русской приставки`
- `И после иноязычной приставки`
- `И в корне (нет приставки)`
- `Ъ разделительный`
- `Ь в корне`
- `Ь в заимствовании`
- `Ъ в сложном слове с числительным`
- `РА- под ударением`
- `РО- без ударения`
- `РАЗ-/РАС- без ударения`
- `РОЗ-/РОС- под ударением`
- `Двойная согласная`
- `ПРА- (родство/древность)`
- `ПРО- (прототип)`
- `ПРО- (противо-)`
- `Неизменяемая: [имя приставки]`
- `Корень (нет приставки)`
- `Словарное`

## Как читать данные (Python)

```python
import pandas as pd

# Читаем Excel
path = r"C:\Users\USER\Documents\kimi\workspace\ege-app\data\atomization\task10\output\task10_atomized.xlsx"
df = pd.read_excel(path)

# Группировка по подкатегориям для персонализации
groups = df.groupby(['main_category', 'subcategory']).agg({
    'id': 'count',
    'correct_letter': 'first'
}).rename(columns={'id': 'count'})

# Статистика по кластерам
clusters = df.groupby('cluster').agg({
    'id': 'count'
}).rename(columns={'id': 'count'})
```

## Как данные используются в приложении

1. **Персонализация** — ученик отвечает на вопросы, система записывает, в каких подкатегориях ошибки
2. **Кластеризация** — если ученик знает "превысить", он скорее знает и "превосходить"
3. **Адаптивность** — больше вопросов из слабых подкатегорий, меньше из сильных
4. **Дерево навыков** — main_category → subcategory → конкретное слово
